# Chapter 2 Story Rules

Applies to: `docs/2-Imaginary-Use-Case/**`

## Title Format
Italicized challenge question as H1:
```markdown
# *"How do we get internet to the community center?"*
```

## Content Style
- Narrative and conversational.
- Frame each section as a problem the protagonist faces.
- Explain why the challenge matters before naming the technology.

## Required Elements
1. **Story context** (2-3 paragraphs): the situation and why it's a problem.
2. **High-level solution overview**: what approach is chosen and why — no implementation steps.
3. **Guide cross-link**:
```markdown
!!! tip "Guide reference"
    For step-by-step instructions, see [Chapter 3 — Topic](../../3-Guide/Topic/index.md).
```
4. WIP marker and TODO if incomplete (see `general.md`).

## Images
- Ch2 images are **narrative and conceptual**: site photos, network topology overviews, scenario illustrations.
- Captions (`<figure>`) are encouraged for photos that set the scene.
- Follow the placeholder workflow and naming convention in `general.md`.

## Do Not
- Write detailed implementation steps (that belongs in Ch3).
